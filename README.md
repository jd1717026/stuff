Task 1 – Analysing the Problem & Designing a Solution (Activity B)

Visual / Interface Designs
Mock designs for the final solution were developed using Figma. The pages planned to be in the solution were all designed, with both desktop and mobile view in mind. The created designs were made for desktop view. The grey blocks represent placeholders that will be swapped for relevant images in the final solution.
The navbar features the created logo for GLH and useful links to other pages. The page links will change depending on if a user is logged in or not. The footer will contain links to social media pages & important notices regarding regulations relating to the farming sector and digital sector. Hero images will be featured on each page to make it clear to the user which page they are on.
Hover effects will be used on different elements of the page. These effects will include underlining / changing the colour of links / buttons when hovered over.
Home Page
The home page is going to feature links to other pages of the solution, such as the login space and the information page. Brief descriptions of what the user can expect to do / see on the site will be featured on this page too.
Log In / Sign Up Page
These pages are going to allow users to create accounts by entering relevant details in the sign-up form. Form validation will be in place, with appropriate error messages being displayed where necessary.
Product Listing Page
Products will be listed on this page in a grid view, which can be amended to a list view depending on screen size. Images of the listed products will be displayed alongside the product name, description and price. A sign in prompt will also be displayed to encourage the user to log in to receive discounts.
Cart Page
This page will display the items the user has added to their cart, giving them the option to add / remove certain items and finalise purchases.
Order Viewing Page
This page will only be available to view if the user is logged in. This page will display the orders the user has previously made. The option to cancel orders will be available.
Information Page
This page will display information to the user about GLH and other important details such as location, contact details, farmer information and sector information.
Dashboard Page
This page will only be available to view if a farmer is logged in. It will take the place of the order viewing page shown to regular customers. It will feature visual displays of stock levels and allow farmers to edit / update stock.
Mobile View Amendments
The cards on the pages will be adjusted to stack on one another vertically & the width of the cards will be reduced to avoid content not fitting on screen. The navbar will feature a burger menu which will appear when the screen size shrinks. The purpose of the menu is to display the links on the navbar vertically rather than horizontally, so users can still see content properly on smaller screens.
Colour Palette
 
The colour palette above was chosen for the solution because the green and browns aligned well with the farm theme as they are typical colours associated with the outdoors. The solution will also feature black + white elements such as text / buttons to comply with WCAG standards by avoiding issues with colour contrast. Both colours were tested on the colour backgrounds and multiple tests resulted in a pass for WCAG AAA. The tests were done on WebAIM: Contrast Checker.
     
Font Style
The font that was chosen for the solution was taken from Google Fonts. The font is called Poppins, and it was chosen due to its simplicity and easiness to read. The font is also free to use as it falls under the Open Font License (OFL), meaning there would be no risk of copyright infringement when using the font.
